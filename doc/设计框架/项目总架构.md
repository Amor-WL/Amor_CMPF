# 概述
# 1.1 设计目标
 * 提供一个通用的多进程框架，支持动态加载业务模块
 * 实现进程间的高效通信机制
 * 支持进程的生命周期管理
 * 提供统一的日志机制
 * 支持高可用性与拓展性
 * 简化业务模块的开发与部署

# 1.2 应用场景
 * 高性能服务器应用
 * 微服务架构
 * 动态更新业务模块的场景
 * 资源密集型应用，需要充分利用多核CPU

# 1.3 设计原则
 * 模块化设计 ：将框架分解为多个独立的组件，便于维护和扩展。
 * 单线程 : 所有阻塞操作应修改为异步，通过事件循环机制实现非阻塞IO。
 * 松耦合 ：组件之间通过明确的接口进行通信，减少相互依赖
 * 高内聚 ：每个组件负责特定的功能，职责明确
 * 可扩展性 ：支持添加新的组件、通信方式和模块类型
 * 可靠性 ：具备容错能力，能够处理各种异常情况
